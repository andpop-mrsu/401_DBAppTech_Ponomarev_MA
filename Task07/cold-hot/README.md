# Игра "Холодно-Горячо"

Веб-версия классической игры "Холодно-Горячо", реализованная на JavaScript с использованием IndexedDB для хранения данных.

## Описание

Игра представляет собой логическую головоломку, где игрок должен угадать трехзначное число с уникальными цифрами. После каждой попытки система выдает подсказки о правильности цифр и их расположения.

## Функциональность

- **Новая игра**: Начало новой игры с вводом имени игрока
- **Список игр**: Просмотр всех сохраненных партий
- **Повтор игры**: Воспроизведение ходов из любой сохраненной партии
- **Автосохранение**: Все игры автоматически сохраняются в базе данных

## Как играть

1. Введите свое имя и нажмите "Начать игру"
2. Попробуйте угадать трехзначное число с уникальными цифрами
3. После каждой попытки вы получите три подсказки:
   - **Холодно** - цифры нет в числе
   - **Тепло** - цифра есть, но не на своем месте
   - **Горячо** - цифра на своем месте
4. У вас есть 10 попыток чтобы угадать число
5. Игра завершается при угадывании числа или исчерпании попыток

## Система подсказок

Подсказки выдаются для каждой цифры отдельно и сортируются в алфавитном порядке для единообразия.

## Пример

- Загаданное число: **123**
- Ваша догадка: **145**
- Результат: **Горячо**, **Тепло**, **Холодно**
   - 1: Горячо (цифра на своем месте)
   - 4: Холодно (цифры нет в числе)
   - 5: Тепло (цифра есть, но не на своем месте)

## Работа с базой данных IndexedDB

Игра использует встроенную браузерную базу данных IndexedDB для сохранения всех партий.

### Сохраняемые данные

Для каждой партии сохраняется:
- **ID игры** - уникальный идентификатор
- **Дата и время** начала партии
- **Имя игрока**
- **Загаданное число**
- **Исход игры** (победа/поражение/в процессе)
- **Все попытки** с результатами проверки

### Структура базы данных

- **Название базы**: `ColdHotGame`
- **Версия**: 1
- **Хранилище**: `games`
- **Индексы**: `date`, `player_name`, `outcome`

### Функциональность базы данных

- **Автосохранение** - каждая новая игра сразу сохраняется в БД
- **Обновление статуса** - при завершении игры обновляется исход и попытки
- **Загрузка истории** - просмотр всех сыгранных партий с сортировкой по дате
- **Воспроизведение** - возможность повторно просмотреть ход любой сохраненной партии

### API базы данных

Класс `Database` предоставляет методы:
- `saveGame()` - сохранение новой игры
- `updateGame()` - обновление данных игры
- `getAllGames()` - получение всех игр
- `getGameById()` - получение игры по ID
